HTML5 Layout

semantic tag란? 태그의 의미와 내용을 서로 맞게 구성해야 한다.

  • header : 사이트 전체의 헤더, article태그의 헤더 역할 시에 사용
  • nav : 네비게이션 표현. (역할만 맞으면 header하위, aside하위, footer하위 다 가능)
  • article : 본문내용, 독립적으로 표현가능한 단위
  • section : 연관된 컨텐츠를 그룹핑 / article 하위도 가능 / 여러 article 묶는 용도
  • aside : 본문내용과 별개인 부수적인 정보 대상. 광고/달력/목록 등
  • footer : 사이트 하단부. 저작권 정보/주소/연락처 등
